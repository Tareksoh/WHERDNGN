Saudi-rule audit of `docs/strategy/` against `Bot.lua`, `Rules.lua`, `BotMaster.lua`, `Net.lua`, `State.lua`, `Constants.lua` at WHEREDNGN v3.0.2.

## TL;DR

- **Rules audited:** 95

- **PARTIAL:** 23 (8 of these are HIGH-impact)
- **MISSING:** 18
- **UNDOCUMENTED:** 0 (most "extras" are actually documented elsewhere)
- **CONTRADICTORY:** 5 (these are the worst)

### Top-5 most-actionable gaps

1. **GAP-01 (CRITICAL/CONTRADICTORY)** — `tahreebClassify` single-event "want" arm: doc says low single discard = "want this suit (no Ace)" (signals.md form 5, 70/25/5 prior), code returns `"hint"` for any non-A non-K-or-T single event. Mirror to v3.0.2 fix: same shape, opposite direction.
2. **GAP-02 (HIGH/MISSING)** — SWA 5+-cards mandatory permission: doc says "5+ cards = mandatory تستاذن" (decision-trees.md:207, endgame.md:191-198), code lets `WHEREDNGNDB.swaRequiresPermission = false` short-circuit even at hand size 5+ (`Net.lua:3256-3270`).
3. **GAP-03 (HIGH/MISSING)** — Hokm trump non-consecutive Takbeer inversion at lead time: decision-trees.md Section 4 row "Takbeer/Tasgheer" + glossary.md says "non-consecutive top trumps → INVERT, lead side first." Wired ONLY in pickFollow trump-winners (Bot.lua:5645-5655); pickLead has no such gate.
4. **GAP-04 (HIGH/MISSING)** — Reverse Al-Kaboot defender-melds-multiplied: doc says "+88 banta + defender's declared melds × meldMult («بالمشاريع» clause)" (saudi-rules.md:120, Constants.lua:204). Rules.lua:1133-1142 sets `meldPoints.B = (sweepTeam == "B") and meldB or 0` and applies `meldMult` — so far so good — but the result is multiplied by `cardMultEffective = K.MULT_BASE` for the 880 raw, while melds use `meldMult` separately. Verified: melds are correctly multiplied, but the "بالمشاريع" wording ALSO implies the offender's melds drop. Code keeps offender melds at 0 (line 1141: only sweepTeam melds awarded). This part is correct. The MISSING piece is: documentation never mentions defender's melds get added to the 88 — the wording is "+88 banta + melds." Code does this correctly. **RE-CLASSIFIED: IMPLEMENTED**. Actual top-5 #4 is GAP-04 below.
4. **GAP-04-replacement (HIGH/PARTIAL)** — Tahreeb 3-discard ascending sequence: decision-trees.md Section 8 three-discard variant says "discard 2 smallest in strict ascending order, keep biggest for last." `tahreebClassify` (Bot.lua:2400-2409) accepts any-length monotonic sequence as ascending/descending, but the SENDER side at Bot.lua:4742-4767 only emits a single low (no ordered sequence-emission across 2+ partner-winning tricks).
5. **GAP-05 (HIGH/PARTIAL)** — Bargiya receiver phase-split: signals.md Section 3 + glossary.md says "≤4 cards: lead immediately; ≥5 cards: burn 1-2 tricks first." Bot.lua:2903-2914 implements the ≥5 downgrade BUT only for `tahreebPrefFlavor == "bargiya"` (confirmed); single-event `bargiya_hint` does NOT receive the same phase-split treatment, and the "void in Bargiya'd suit → lead it anyway" exception (signals.md:198) is missing entirely.
