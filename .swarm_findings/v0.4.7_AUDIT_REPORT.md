## 2. Critical Bugs Fixed

### C-1 — Net.lua authority gate rejected host-signed plays for human seats

**File:** `Net.lua:1104`

The v0.4.6 turn-desync fix added a `fromHost` escape so that bot plays signed by the host could pass the `s.turn ~= seat` gate. The escape was written as `if isBot(seat) and fromHost then` — it only applied to bot seats. Human AFK auto-plays are also signed and sent by the host on behalf of the human seat; they hit the same gate, failed the `isBot` check, and were silently dropped. The client remained stuck showing the pre-autoplay state. Fixed by changing the condition to `if fromHost then`, making the escape cover any host-authored play regardless of seat type. The existing idempotence guard at line 1108 prevents double-apply.

**Patch applied:** `Net.lua:1104` — condition `isBot(seat) and fromHost` changed to `fromHost`.

### C-2 — _HostTurnTimeout silently forfeited melds for AFK humans in trick 1

**File:** `Net.lua` (HostTurnTimeout function)

When an AFK human timed out during trick 1, `_HostTurnTimeout` auto-played a card on their behalf but did not auto-declare their melds. The Saudi meld window is trick 1 only; once trick 1 ends, any undeclared meld is permanently forfeited per Saudi rule. A 50-point Quarte (worth 100 gp in a Bel contract) was silently lost. `MaybeRunBot` had the correct meld-declaration logic; `_HostTurnTimeout` was missing the parallel call. Fixed by mirroring the `S.ApplyMeldDeclaration` call sequence from `MaybeRunBot` at the top of `_HostTurnTimeout`'s human-seat branch.

**Patch applied:** Meld auto-declaration block mirrored from `MaybeRunBot` into `_HostTurnTimeout`.

### C-3 — BotMaster fallback deal path missing meldPins

**File:** `BotMaster.lua` (fallback rollout path)

The primary rollout deal path pinned declared meld cards to their owning seat via `meldPins` before shuffling remaining cards. The fallback path (triggered when the primary path's constraint-solving timed out) skipped `meldPins` entirely and shuffled all 32 cards together. Declared meld cards could scatter across all 4 seats in the rollout world-state, corrupting every downstream EV calculation for rounds with active melds. Fixed by inserting the same `meldPins` construction block used by the primary path at the top of the fallback shuffle.

**Patch applied:** `meldPins` construction block added to `BotMaster.lua` fallback path, mirroring primary path.

### H-1 (was HIGH) — SWA timer fired during paused games

**File:** `Net.lua` (`_OnSWAReq` and `LocalSWA`)

Both the `_OnSWAReq` C_Timer.After callback and the `LocalSWA` auto-approve timer fired unconditionally. If the game was paused (e.g. mid-pause host migration, or a reconnect pause) when the SWA request arrived, the 5-second window ran down and auto-approved while play was suspended. An SWA resolved while players were not watching their screens. Fixed by adding a `isPaused` check inside both timer callbacks; if paused, the timer re-arms for the remaining duration once `RESUME` fires.

**Patch applied:** Pause guard + re-arm-on-resume added to both SWA timer sites.

### H-2 (was HIGH) — Bot.OnPlayObserved fired on replay frames

**File:** `Bot.lua` (`OnPlayObserved`)

During late-joiner resync the host replays prior trick cards as MSG_PLAY frames flagged `isReplay = true`. `Bot.OnPlayObserved` had no guard for this flag. Replayed frames incremented `trumpEarly`/`trumpLate`, set `firstDiscard`, and updated void-inference as though the cards were being played live. A bot joining mid-game started round 3 with corrupted memory state populated by re-processed tricks from rounds 1 and 2. Fixed by adding `if isReplay then return end` at the top of `OnPlayObserved`.

**Patch applied:** `not isReplay` guard added at `Bot.lua` `OnPlayObserved` entry.

## 3. Concerns Identified — Medium

### M-1 — Bot.PickKawesh missing explicit 5-card hand count check

**File:** `Bot.lua` (`PickKawesh`)  
**Rationale:** Kawesh is only valid when a player holds exactly 5 cards of one suit. `PickKawesh` relies on the phase gate (only called in PHASE_DEAL before any cards are played) to ensure hand sizes are correct. If `PickKawesh` were ever called from a different phase (debug, test harness, future refactor), it could evaluate an incomplete hand and incorrectly signal Kawesh. The phase gate is a soft contract, not an assertion.  
**Recommended action:** Add `if #suitCards ~= 5 then return nil end` guard at the top of the suit-eval loop. One line, zero risk.

### M-2 — WHEREDNGN.lua no nil-guard on B.Net at CHAT_MSG_ADDON dispatch

**File:** `WHEREDNGN.lua:259`  
**Rationale:** `B.Net` is assigned during `ADDON_LOADED`. Between `ADDON_LOADED` and `PLAYER_LOGIN`, the CHAT_MSG_ADDON event can fire (party-channel messages from a previous session draining). At that point `B.Net` is nil and line 259 (`B.Net:OnMessage(...)`) will throw a nil-index error, generating a Lua error in the user's chat. Unlikely in practice but reproducible by logging in with an existing party already formed.  
**Recommended action:** `if B.Net then B.Net:OnMessage(...) end` at line 259.

### M-3 — WHEREDNGN.lua no pcall around critical dispatch paths

**File:** `WHEREDNGN.lua` (event dispatch)  
**Rationale:** Any unhandled Lua error in the CHAT_MSG_ADDON handler silently breaks the WoW addon for the session without player-visible feedback (WoW swallows addon errors). Given that Net.lua is 3617 lines with complex state, a nil-index in any sub-path terminates all subsequent message processing. A pcall wrapper with `B.Log.Error` on failure would convert silent corruption into a visible log entry.  
**Recommended action:** Wrap the body of the CHAT_MSG_ADDON handler in `pcall`; log the error trace.

### M-4 — UI.lua renderActions has no localSeat guard for spectators

**File:** `UI.lua` (`renderActions`)  
**Rationale:** `renderActions` builds the action button row using `localSeat` to determine which actions are valid. If `localSeat` is nil (spectator or pre-deal state), several table-index operations access `S.s.hands[localSeat]` without a nil-check. This does not crash because the early-return `if not S.s` guard fires first in most paths, but if `S.s` is populated and `localSeat` is nil (spectator watching an in-progress game), the function reaches the hand-access without a safety net.  
**Recommended action:** Add `if not localSeat then return end` immediately after `if not S.s then return end`.

### M-5 — Slash.lua /baloot host command blocked from PHASE_LOBBY and PHASE_SCORE

**File:** `Slash.lua` (`/baloot host` handler)  
**Rationale:** The spec allows the host to initiate a new round from PHASE_SCORE (end-of-round summary) and to configure the game from PHASE_LOBBY. The current slash handler has `if phase ~= PHASE_PLAY then return end` which rejects both. This is a spec divergence: host management commands that should work in lobby or score phases silently do nothing, with no feedback to the player.  
**Recommended action:** Audit the full intent of each `/baloot host` subcommand and gate them individually rather than using a single phase blanket.

### M-6 — Cards.lua C.Shuffle has shared LCG state (coroutine safety)

**File:** `Cards.lua` (`C.Shuffle`)  
**Rationale:** `C.Shuffle` advances a module-level LCG seed in place. If two coroutines call `C.Shuffle` concurrently (not currently the case, but BotMaster rollouts are structured as iterative loops that could be refactored to coroutines), the shared seed state yields non-independent shuffle sequences. This is low risk today but is a latent trap.  
**Recommended action:** Pass seed as parameter or use `math.random` with per-call seeding. Non-urgent; flag in a comment at minimum.
